# 智能语音服务学习笔记

> 以 Chenxing Agent ROS (Daystar Bot) 为工程锚点，系统学习现代语音模型的**信号处理理论、模型架构、边缘部署优化**
> 师傅：OpenCode AI · 学徒：用户

---

## 学习计划总览

> **定位调整**：第一版大纲过于围绕工程管道（ALSA 路由、ROS 2 中间件、Loopback 配置），属于软件架构范畴——用户已具备。第二版核心转向**语音模型的数学基础、架构演进、边缘推理优化**。
>
> 本课程的表层载体仍是 Chenxing Agent ROS 中的具体模型（GTCRN、Zipformer、Kokoro 等），但每个专题都会从**第一性原理**出发：特征怎么来的？损失函数为什么这样设计？流式推理和非流式推理在数学上差了什么？量化到底损失了什么精度？
>
> 重点倾斜：**边缘侧模型理论 + 部署优化**——因为当前工程就是纯端侧（Jetson Orin NX），所有模型必须在有限算力/内存下运行。

## 后续学习方式

### 1. 纵向主线 + 按需补课

核心纵向主线：**特征提取 → ASR 架构演进（CTC→RNN-T→Attention→Whisper）→ 量化部署优化**。这个链条上任何缺口（如 Transformer 理解不深、概率图模型不熟）都会围绕当时的具体问题补充，不脱离语音任务去系统地学 NLP 或信号处理整门课。

需要贯穿补全的通用理论能力：

- **概率图模型**：HMM（forward-backward）、CTC 的 alignment lattice、WFST 的 semiring
- **最优化理论**：CTC loss 的梯度推导、LF-MMI 的分子/分母图、RNN-T 的 pruned loss
- **信息论**：熵、交叉熵、KL 散度——贯穿所有 loss 函数的设计
- **Transformer 深入**：在 Course 7 集中突破，后续持续加深（KV cache、flash attention 等）
- **编译原理基础**：WFST 里的 compose/determinize/minimize、ONNX 图优化 pass——本质上都是编译器技术

### 2. 每课形成理论与实践闭环

与嵌入式课程一致：

1. **数学理解**：关键公式手推一遍（CTC forward-backward、attention 的 Q·K^T/√d_k）
2. **代码走读**：在 sherpa-onnx 或本项目源码中找到对应的实现路径
3. **最小实验**：做一个参数范围的修改并观察输出变化（如改变 beam size、量化位宽）
4. **观测证据**：保存识别结果、延迟数据或精度对比
5. **结论沉淀**：记录到相应的学习记录

### 3. 分层能力记录

| 层级 | 含义 |
|------|------|
| **已理解** | 能手推关键公式（如 CTC 前向后向），能向别人讲清楚 |
| **已走读** | 在源码中找到实现路径，理解每个函数的输入输出和数据流 |
| **已实验** | 修改参数/替换模型后完成端到端测试并对比结果 |
| **已优化** | 成功对某个模块做量化/剪枝/模型替换，并获得可衡量的精度-延迟 tradeoff |
| **已沉淀** | 将可复用的脚本、checklist 或性能数据写回知识库 |

### 4. 工程锚点使用原则

Chenxing Agent ROS 项目在课程里的角色是**具体实例**而非**教学内容**：

- 文中出现 "本项目" 均指 chenxing_agent_ros
- 每种架构/理论讲完后，回到本项目找到对应的实现（如 CTC → Zipformer encoder 的 chunk 流式模式）
- 每个量化/优化专题结束后，在本项目模型上做验证实验

## 术语表 (Glossary)

### 信号处理

| 术语 | 定义 |
|------|------|
| **Nyquist 采样定理** | 采样率 ≥ 2×信号最高频率，才能无失真重建。语音通常 8kHz（电话）/16kHz（宽带） |
| **STFT** | Short-Time Fourier Transform：加窗分段做 DFT，得时间-频率二维表示 |
| **频谱泄漏** | 非整周期截断导致的频域能量扩散，通过窗函数（Hamming/Hann）抑制 |
| **梅尔尺度 (Mel Scale)** | 模拟人耳非线性频率感知：$m = 2595 \log_{10}(1+f/700)$，低频分辨率高，高频分辨率低 |
| **MFCC** | Mel-Frequency Cepstral Coefficients：FBank → log → DCT → 取前 12-13 个系数，去相关+降维 |
| **FBank** | Filterbank 能量特征：保留全频带信息，比 MFCC 多相关性（DNN 更喜欢 FBank） |
| **动态特征** | Δ（一阶差分）和 ΔΔ（二阶差分），捕捉时间变化趋势 |

### ASR 架构

| 术语 | 定义 |
|------|------|
| **CTC** | Connectionist Temporal Classification：引入 blank token 解决对齐问题，用前向后向算法计算 marginalization loss。天然适合流式，但假设帧间独立（需语言模型纠正） |
| **RNN-T** | Recurrent Neural Network Transducer：Prediction Network（语言模型）+ Encoder（声学模型）+ Joint Network。比 CTC 强在不用外部语言模型建模上下文 |
| **LAS** | Listen-Attend-Spell：encoder 听整句 → attention 对齐 → decoder 逐词生成。效果好但不流式 |
| **WFST** | Weighted Finite-State Transducer：Kaldi 的核心数据结构。HCLG = H∘C∘L∘G，一个图同时表达声学-发音-语言三层约束 |
| **LF-MMI** | Lattice-Free Maximum Mutual Information：Kaldi chain model 的序列级区分性训练准则，替代了传统的交叉熵帧级训练 |
| **Zipformer** | 新一代流式 ASR encoder（sherpa-onnx 主力）：改进 downsampling + attention 结构，比 Conformer 更快 |

### KWS

| 术语 | 定义 |
|------|------|
| **虚警率 (FAR)** | False Alarm Rate：每小时误唤醒次数。消费级产品通常要求 < 1 次/小时 |
| **漏检率 (FRR)** | False Reject Rate：唤醒词说出但未被检测到的比例 |
| **TC-ResNet** | Temporal Convolution + ResNet：轻量 KWS 的经典架构，参数量 < 300K |

### TTS

| 术语 | 定义 |
|------|------|
| **声码器 (Vocoder)** | 将声学特征（Mel 谱）还原为波形的模型。传统：Griffin-Lim；神经：WaveNet/WaveGlow/HiFi-GAN |
| **自回归 TTS** | 逐采样点/逐帧生成，质量高但慢（Tacotron2+WaveNet） |
| **非自回归 TTS** | 并行生成，速度快（FastSpeech 用 duration predictor 解决长度 mismatch） |
| **VITS** | VAE + Flow + GAN 的端到端 TTS：隐变量建模韵律变化，不需要单独的声码器 |
| **Duration Predictor** | 预测每个音素对应的帧数，非自回归 TTS 的关键组件 |

### 边缘部署

| 术语 | 定义 |
|------|------|
| **PTQ** | Post-Training Quantization：训练后量化，不需要重新训练，但精度损失可能较大 |
| **QAT** | Quantization-Aware Training：训练时模拟量化噪声，精度更高但需要训练管线 |
| **对称/非对称量化** | 对称：zero-point=0，范围 [-127,127]；非对称：有 zero-point，如 [0,255]，适合 ReLU 输出 |
| **per-tensor / per-channel 量化** | per-tensor：整层共用一个 scale；per-channel：每个输出通道独立的 scale（更精细，常用于卷积权重） |
| **EP** | Execution Provider：ONNX Runtime 的硬件后端抽象（CPU/CUDA/TensorRT/CoreML...） |
| **Graph Optimization** | ONNX Runtime 在加载模型时自动进行的图变换：常量折叠、算子融合（如 Conv+BN→Conv）、冗余节点消除 |
| **DLA** | Deep Learning Accelerator：Jetson 上的专用推理硬件，功耗远低于 GPU，适合 KWS 等常驻任务 |

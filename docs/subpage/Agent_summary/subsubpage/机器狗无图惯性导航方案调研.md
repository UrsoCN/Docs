# 机器狗（四足机器人）上的无图惯性导航方案综述

> 与 AI-IO 类似的、**无需视觉/LiDAR 地图、仅用本体传感器**的惯导方案调研。大部分代码已开源。

---

## 一、X-IONet (RA-L 2026) ⭐ 最推荐

**论文：** [X-IONet: Cross-Platform Inertial Odometry Network for Pedestrian and Legged Robot](https://arxiv.org/abs/2511.08277)
**作者：** Dehan Shen, Changhao Chen 等
**关键词：** 单 IMU、跨平台、双阶段注意力、EKF 融合

**核心：**
- **仅用单一 IMU** 做惯导——不依赖任何关节/视觉信号
- 采用「规则基专家选择模块」区分行人/四足机器人运动模式，路由到对应专网
- **双阶段注意力：** Stage 1 捕捉时序依赖，Stage 2 捕捉轴间相关性
- 输出位移 + 不确定性 → 经 EKF 滤波得到鲁棒位姿

**精度：**

| 数据集 | ATE 降幅 | RTE 降幅 |
|---|---|---|
| RoNIN (行人) | 14.3% | 11.4% |
| GrandTour (四足) | 11.8% | 9.7% |
| **宇树 Go2 (自采)** | **52.8%** | **41.3%** |

**对机器狗的意义：** 直接跑在 Go2 上验证过，纯 IMU 方案精度很高。缺点是不利用关节编码器，有点浪费已有传感器。

## 二、Buchanan et al. (CoRL 2022) — 经典基线

**论文：** [Learning Inertial Odometry for Dynamic Legged Robot State Estimation](https://arxiv.org/abs/2111.00789)
**作者：** Russell Buchanan, Marco Camurri, Frank Dellaert, Maurice Fallon (牛津/CMU/Edinburgh)
**机器人：** ANYmal (四足)
**关键词：** 学习型 IMU 位移测量、紧耦合关节编码器、因子图/滤波

**核心方法：**
1. **输入：** IMU (acc+gyro) + 关节编码器读数
2. **网络：** TCN（时序卷积）从 IMU 历史窗口预测**位移增量** (δx, δy, δz) + 对角协方差矩阵
3. **融合：** 两种方式均可：
   - **EKF**：IMU 传播（400Hz）+ 网络更新（~20Hz）
   - **因子图**：VILENS 风格的滑动窗口因子图
4. **关节编码器作用：** 提供运动学约束（足端接触已知时）或作为因子图中的独立速度因子

**精度：**
- 在 ANYmal 上比纯运动学-惯性融合降低 **37%** 位姿误差
- 户外越野地形、打滑、楼梯等场景均有测试

**开源：** 代码在 [uzh-rpg/learned_inertial_model_odometry](https://github.com/uzh-rpg/learned_inertial_model_odometry) 基础上改造——恰好是 AI-IO 也引用的代码库。

**与 AI-IO 对比：**

| 对比项 | Buchanan 2022 | AI-IO (ICRA 2026) |
|---|---|---|
| 载体 | ANYmal 四足 | 四旋翼 |
| 辅助信号 | 关节编码器 | 旋翼转速 |
| 网络 | TCN (ResNet风格) | Transformer + 模态编码 |
| 输出 | 位移增量 + 协方差 | 体速度 + 协方差 |
| 融合 | EKF / 因子图 | MSCKF EKF |

## 三、Legolas (CMU 2025) ⭐ 最创新

**主页：** [learned-odom.github.io](https://learned-odom.github.io/)
**作者：** Justin Wasserman, Ananye Agarwal, Rishabh Jangir, Deepak Pathak, Abhinav Gupta (CMU / UIUC)
**关键词：** 纯数据驱动、零真实数据训练、端到端

**最大亮点：** **仅用仿真数据训练，直接在实体机器狗上 zero-shot 泛化。**
- 不需要任何真实轨迹标注（动捕/VICON）
- 不需要分析模型（无需手动调参）
- 在宇树 A1 和 Unitree Go2 上均验证

**方法：**
1. **输入：** IMU (6轴) + 关节位置/速度 + 足端接触状态
2. **模型：** 小 Transformer（因果自注意力），在仿真中收集的各种步态数据训练
3. **输出：** 瞬时速度 + 协方差（每个时间步）
4. **EKF 融合：** 将网络输出与 IMU 传播通过 EKF 融合得到 6-DoF 位姿

**精度：**
- 比分析滤波方案 **RPE 降低 73%**
- 比真实行为克隆方案 **RPE 降低 87.5%**
- 在 180m 户外场景下漂移很小
- 楼梯、室内、室外、草地均验证通过

**代码：** 已开源（repo 链接在项目主页）

## 四、OCELOT (arXiv 2026.05) — 最新轮式里程计

**论文：** [OCELOT: Odometry and Contact Estimation for Legged Robots](https://arxiv.org/abs/2605.21863)
**作者：** Emre Girgin, Cagri Kilic
**关键词：** 误差状态 EKF、接触检测、打滑识别、开源 ROS2

**核心：**
- **纯本体传感器：** IMU + 关节编码器 + 力传感器
- 滤波器由**被检测为静止支撑的足端**进行更新
- **关键创新：** 融合两个并行接触检测器：
  - 基于力传感器 GMM 的有限状态机（物理接触确认）
  - 基于运动学 GLRT 的足端速度估计（运动学静止验证）
  - 两者的融合评分同时判断接触与否和不确定性
- 显式识别和拒绝打滑

**数据：** 29 条序列，2.4 km，覆盖混凝土/草地/碎石/岩石
**开源：** ✅ 代码 + ROS2 包

## 五、IMM-Proprio (arXiv 2026.03)

**论文：** [Interacting Multiple Model Proprioceptive Odometry for Legged Robots](https://arxiv.org/abs/2603.29383)
**关键词：** 交互式多模型 (IMM)、足端里程计与 IMU 的模态切换

**核心：** 利用 IMM 框架在不同运动模式（静态站立/走/跑/跳跃/打滑）之间主动切换融合策略，适合需要在多种步态间切换的机器狗。

---

## 六、Multi-IMU 融合 (arXiv 2025.07)

**论文：** [Multi-IMU Sensor Fusion for Legged Robots](https://arxiv.org/abs/2507.11447)
**作者：** Shuo Yang 等
**关键词：** 多 IMU、紧耦合 IMU+关节、因子图

**核心：**
- 多 IMU（多个部位安装）+ 关节编码器 → EKF 融合速度
- 叠加视觉相机 → 因子图滑动窗口估计（VIL 架构）
- 可视为 Buchanan 方法的"多 IMU + 视觉版"

---

## 七、MUSE (Real-Time Multi-Sensor, IRI 2025)

**主页：** [arxiv.org/html/2503.12101](https://arxiv.org/html/2503.12101v1)
**关键词：** 姿态观测器 + 打滑检测 + 足端里程计 + 多模态融合框架

**架构：**
```
姿态观测器 (IMU)
     + 打滑检测 (力/运动学)
     + 足端里程计 (关节+接触)
     + 相机/LiDAR 里程计
     → 扩展卡尔曼滤波 → 完整状态估计
```

支持可选剔除视觉/LiDAR，退化为纯本体方案。

## 八、方案对比总结

| 方案 | 传感器 | 是否纯本体 | 是否需要真实标注 | 代码 | 关键特征 |
|---|---|---|---|---|---|
| ✳️ **X-IONet** | **单 IMU** | ✅ | ✅ 需要动捕数据 | 未知 | **真正纯 IMU**，Go2 验证 |
| ❤️ **Legolas** | IMU+关节+接触 | ✅ | ❌ **仅仿真** | ✅ | 零真实数据，泛化最强 |
| ⭐ **Buchanan** | IMU+关节 | ✅ | ✅ 需要动捕 | ✅ | 经典基线，EKF+因子图双模态 |
| 🔥 **OCELOT** | IMU+关节+力 | ✅ | ❌ (不需学习) | ✅ ROS2 | 打滑识别最强+开源 |
| 🔥 **IMM-Proprio** | IMU+关节+力 | ✅ | ❌ (不需学习) | 未知 | 多步态切换自适应 |
| 🀄 **MUSE** | IMU+关节+力+视觉 | 可选纯本体 | ✅ 或 分析模型 | unknown | 框架灵活，足端里程计为核心 |
| 🔁 **AI-IO** | IMU+旋翼转速 | 是(对四旋翼) | ✅ VICON | ✅ | 可为机器狗改造（关节替代转速） |

## 对机器狗项目的实际建议

### 推荐方案

根据你的机器狗产品需求（前进 X 米/转向 Y 弧度），按优先级：

```
🥇 Legolas (零标注+仅仿真+Go2 验证) — 下限最高
🥇 OCELOT (ROS2+打滑识别+纯分析) — 稳定可靠
🥇 X-IONet (纯 IMU、简洁) — 上限最高但需动捕
🥈 Buchanan (经典方案+开源) — 折中
```

### 精度预期

| 动作 | Legolas 预期 | OCELOT 预期 | X-IONet 预期 |
|---|---|---|---|
| 前进 5m | ~10-20cm | ~10-20cm | ~20-40cm |
| 前进 20m | ~40-80cm | ~30-60cm | ~70-150cm |
| 转向 90° | ~3-5° | ~2-4° | ~5-10° |
| 打滑地面 | 较好（学习型） | 最好（显式检测） | 中等 |
| 长距>100m | ~5-10% 距离 | ~3-8% 距离 | ~10-15% 距离 |

### 实现路径

```
快速原型：    接 Legolas + 数据预处理管道
高精度部署：  接 OCELOT (纯分析) + Legolas 网络 (学习型) 双模式
纯 IMU 方案： 接 X-IONet + 微调 (需自采动捕数据)
```
